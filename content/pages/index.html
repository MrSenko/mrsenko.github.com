What software is currently supported

How does it work?

  1. Report a problem to Mr. Senko;
  2. Mr. Senko will prioritize, reproduce, test and fix the problem;
  3. Mr. Senko will push the changes to GitHub and create a new package version;
  4. You download and upgrade to the latest version!

What is 1 subscription == 10 packages?

One subscription provides you with technical support for a maximum of 10 software packages (aka open source projects). You can select any 10 from the list of supported software. If you need support for more packages you have to increase the number of your subscriptions.

What is 1 subscription == 1 priority vote?

Every Monday morning Mr. Senko decides what bugs and feature requests will be included in the next package version. This is based on their severity and demand from developers. 1 subscription counts as 1 vote! If you are developing multiple applications using the same library and want an issue to be fixed faster you have to increase the number of your subscriptions! Mr. Senko considers it a good practice to have 1 subscription per project!

How do I download packages from Mr. Senko?

Your subscription maps to a private URL which you can use with your package management tools. You can use the same URL within multiple development projects (e.g. front-end and back-end). If your subscription doesn't provide a private repository then you have to wait for the open source community to accept the changes proposed by Mr. Senko and release a new version on their website!
